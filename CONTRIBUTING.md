# Contributing to GlyphAtlas

First off, thank you for considering contributing to GlyphAtlas!

## Project Status: Feature-Complete

As of `v1.0.0`, GlyphAtlas is considered **feature-complete**. The primary goal of this tool is to provide a simple, lightweight, and focused utility for extracting Unicode codepoints for C/C++ projects. To maintain this simplicity and prevent feature creep, we are **not** actively looking to add major new features or significant UI overhauls.

## What we accept

While major new features are unlikely to be merged, we still welcome contributions in the following areas:

- **Bug Fixes:** Fixing crashes, incorrect codepoint extractions, UI glitches, or platform-specific issues.
- **Performance Improvements:** Optimizations that make the tool faster or more efficient without adding complexity.
- **Documentation:** Fixing typos, improving instructions, or clarifying the documentation.
- **Refactoring:** Minor code cleanups that improve readability and maintainability.

## How to Contribute

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/your-username/glyph-atlas.git
   ```
3. **Create a new branch** for your fix or improvement:
   ```bash
   git checkout -b fix-issue-name
   ```
4. **Make your changes**, ensuring they align with the project's goals.
5. **Test your changes** to verify they work as expected.
6. **Commit and push** your branch:
   ```bash
   git commit -m "Brief description of the fix"
   git push origin fix-issue-name
   ```
7. **Open a Pull Request** against the `main` branch of the original repository. Please clearly describe what your PR fixes or improves.

Thank you for helping keep GlyphAtlas robust and reliable!
